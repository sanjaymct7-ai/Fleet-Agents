## 2. FIXED ARCHITECTURAL DECISIONS
- **Blackboard pattern:** agents NEVER call each other; they coordinate exclusively through DB state. Order/route/exception `status` fields choreograph handoffs (e.g., dropped orders stay `new` → next planner run sweeps them up).
- **One writer per table:** each table has exactly one component allowed to write it.
- **LLM/code division:** LLMs only extract (Intake), judge from a FIXED menu (Recovery), and write within templates (Comms) or summarize given stats (Reporting). Deterministic code does all validation, geocoding, math, and state mutation. LLM never geocodes and never free-writes to customers.
- **Propose → gate → execute separation** (structure retained even after the pivot below): Recovery writes `proposals`; `risk_tier()` (pure rules, never LLM) tiers them; `executor.py` (no LLM) applies them.
- **PIVOT D8 — HITL removed, fully autonomous:** all tiers auto-execute; Recovery menu = `["notify_delay","reschedule_next_day"]`; `no_driver` exceptions log-and-resolve as staffing notes; enterprise failed deliveries auto-reschedule with enterprise tier treated specially. Risk tiers survive as visibility flags only. `manager_agent.py`, `escalation_graph.py`, and the Command Center Approvals panel are DEAD/IDLE code awaiting cleanup.
- **Two-door system (NEW THIS SESSION):** 
  - **Manager portal** (pages/4_manager_portal.py) — see ALL orders, ALL vehicles, live fleet map, analytics, settings.
  - **Customer portal** (pages/5_customer.py) — see ONLY your orders, track YOUR van in real-time, get YOUR messages, submit feedback. Pseudo-login (customer name) teaches data scoping discipline: every query filters by `customer_name`.
  - In production, Supabase Auth + RLS would enforce this. Here it's a pedagogical exercise.
- **Order priority REMOVED (NEW THIS SESSION):** All orders are standard. No tiering by urgency (1-5 scale). Simplifies solver, simplifies UI, clarifies focus.
- **Customer tiers simplified to 2 ONLY (NEW THIS SESSION):** `standard` (individual customers) and `enterprise` (company accounts). Removed `premium` tier entirely. Updated schema constraints, LLM prompts, agent logic.
- **Real addresses + Geoapify (NEW THIS SESSION):** 
  - Old: hardcoded `PLACES` dict (10 locations)
  - New: customers type real addresses → `src/geocoding.py` queries Geoapify → lat/lng stored
  - Why: production-ready, works anywhere, no maintenance of place lists
  - Intake agent LLM extracts `pickup_address` and `drop_address` (text), validation geocodes them
- **Live manager map (NEW THIS SESSION):** Manager Portal has new "🗺️ Live Map" tab showing all vehicle_positions as Folium markers with zoom, fit, satellite tiles. Shows ONLY all vehicles (manager view). Customer portal shows mini-map with ONLY their van + drop point (customer view).
- **DEFERRED-TO-END (decision D9):** Supabase Auth + `profiles` + `orders.customer_id` + RLS + service-role/anon client split. Spec agreed (self-signup, profile editing, auth-gated order/track). DO NOT enable RLS or copy the service_role key before that phase.
- **Depot model routing:** vehicles deliver drop-points from one depot (`DEPOT` in `matrix.py`); true pickup-and-delivery is a declared stretch goal, not built.
- Fleet is data: `vehicles` table (heterogeneous capacities) feeds the solver; `N_VEHICLES` constants are gone.

## 3. SYSTEM CONVENTIONS & STYLING
- LLM JSON protocol: prompt demands "ONLY JSON"; parse = strip ```json fences → `json.loads` → gate result against the allowed menu/keys → on failure, leave work item for retry (never crash, never accept out-of-menu actions).
- DB deletes must follow FK order (children first): notifications → proposals → exceptions → vehicle_positions → route_stops → routes.
- Correctness pushed into the schema: CHECK constraints on every status/enum, UNIQUE constraints as dedupe rails (`exceptions(type, order_id)` + partial unique index `(type, route_id) WHERE order_id IS NULL`).
- Timestamps stored ISO/timestamptz; solver works in **minutes-since-midnight** ints; driver shifts are `'HH:MM'` text.
- Streamlit: `@st.cache_resource` for singletons; `st.form` for multi-field inputs; `st.session_state.running_day` guards autorefresh during the synchronous Run-day; page files in `pages/` are lowercase (Linux hosting is case-sensitive — burned once).
- Print-based logging with tags (`[llm] answered by: groq`, `[matrix] ... 0 credits`); cost printed BEFORE any credit-spending API call.
- Git: commit per working step with descriptive message; `git rm` only works on tracked files; never commit secrets (revoke+reissue if leaked, deleting the commit is not enough).

## 5. DISCOVERED BUGS & SOLUTIONS
- **Solver `NO SOLUTION`:** fleet capacity had zero headroom vs total demand (bin-packing needs slack) → give ~20–30% headroom; heaviest single order must fit one vehicle. "No solution" is an ANSWER, not an error.
- **7 orders silently vanished:** dispatch found no eligible driver, route stayed unassigned, simulator skipped it → fix: dispatch raises `no_driver` exception + partial unique index for route-level dedupe. Diagnosed via KPI arithmetic (20+3 ≠ 30) — check data before blaming UI.
- **`PGRST205 table not found`:** SQL step skipped → create table; generally means schema drift.
- **Geocode hallucination risk:** LLM must NEVER output coordinates (plausible-but-wrong) → address lookup + reject unknowns.
- **Null-island bugs:** placeholder `(0.0, 0.0)` in DEPOT/PLACES/bbox silently poisons data → always replace before running.
- **`git rm` "pathspec did not match":** file was never tracked (use `del`) or case mismatch (`2_Approvals` vs `2_approvals`) — git is case-sensitive, Windows isn't.
- **"not a git repository":** wrong cwd (a `cd` line got glued to the next command).
- **Streamlit reruns killing state:** graph/checkpointer must live in `@st.cache_resource`; autorefresh must pause during Run-day via session_state flag or it kills the run.
- **`streamlit_autorefresh` ModuleNotFound on cloud:** dependency missing from requirements.txt — every new pip install needs a requirements line.
- **Geoapify quota:** matrix costs max(n,n)×min(n,10) credits (31×31=310, NOT 1/call) → Postgres cache mandatory; 2nd run must print "0 credits".
- **ScriptRunContext warning** in bare scripts: harmless (st.secrets outside app).
- **Supabase free tier:** pauses after ~7 idle days (click Resume); NO backups — seed scripts are the backup.
- **reset re-run FK errors:** delete children before parents (order listed in §4).
- **vehicle_positions column name:** was `lon`, corrected to `lng` (2024-07-08) to match Geoapify output order [lng, lat].
- **Priority field removed:** all references deleted from orders table, solver, recovery_agent, comms_agent. No migration needed (field never stored customer-visible priority logic).
